# Python 코드 컨벤션

## 네이밍 규칙

### 변수명
- snake_case를 사용해야 합니다
- 소문자와 언더스코어로만 구성합니다
- 의미 있는 이름을 사용해야 합니다
- 단일 문자 변수는 루프 카운터(i, j, k)를 제외하고 사용하지 않습니다

### 함수명
- snake_case를 사용해야 합니다
- 동사로 시작해야 합니다
- 명확한 동작을 표현해야 합니다
- 축약어 사용을 지양합니다

### 클래스명
- PascalCase를 사용해야 합니다
- 명사를 사용해야 합니다
- 단수형으로 작성합니다

### 상수
- UPPER_CASE를 사용해야 합니다
- 언더스코어로 단어를 구분합니다
- 모듈 레벨에서 정의합니다

### Private 멤버
- 이름 앞에 언더스코어 하나(_)를 추가합니다
- 외부에서 직접 접근하지 않아야 합니다

### 모듈명
- 짧고 소문자로만 작성합니다
- 언더스코어 사용을 최소화합니다

## 코딩 스타일

### 들여쓰기
- 4개의 공백을 사용해야 합니다
- 탭 문자를 사용하지 않습니다

### 줄 길이
- 한 줄은 최대 79자를 넘지 않아야 합니다
- 주석과 docstring은 최대 72자를 넘지 않습니다
- 긴 줄은 괄호나 백슬래시로 적절히 분리합니다

### 빈 줄
- 최상위 함수와 클래스 정의는 2줄의 빈 줄로 구분합니다
- 클래스 내부 메서드는 1줄의 빈 줄로 구분합니다
- 논리적 섹션을 구분할 때 빈 줄을 사용합니다

### 공백
- 연산자 앞뒤로 공백 1개를 사용합니다
- 콤마 뒤에는 공백 1개를 사용합니다
- 괄호 안쪽에는 공백을 사용하지 않습니다
- 함수 인자의 기본값 할당 시 = 앞뒤로 공백을 사용하지 않습니다

### Import
- 표준 라이브러리, 서드파티 라이브러리, 로컬 모듈 순서로 그룹화합니다
- 각 그룹 사이에는 빈 줄을 추가합니다
- 와일드카드 import(from module import *)는 사용하지 않습니다
- 한 줄에 하나의 import만 작성합니다

### 문자열
- 작은따옴표 또는 큰따옴표를 일관성 있게 사용합니다
- 여러 줄 문자열은 삼중 따옴표를 사용합니다
- f-string을 사용하여 문자열 포맷팅을 수행합니다

### 주석
- 코드와 모순되는 주석은 작성하지 않습니다
- 완전한 문장으로 작성합니다
- # 뒤에 공백 1개를 추가합니다
- 코드 위에 작성하며, 같은 줄에 작성하지 않습니다

### Docstring
- 모든 공개 모듈, 함수, 클래스, 메서드에 작성합니다
- 삼중 큰따옴표를 사용합니다
- 한 줄 요약으로 시작합니다
- 복잡한 함수는 Args, Returns, Raises 섹션을 포함합니다

## 코드 구조

### 함수
- 하나의 함수는 하나의 기능만 수행해야 합니다
- 함수는 최대 50줄을 넘지 않도록 합니다
- 중첩 깊이는 최대 4단계를 넘지 않습니다

### 클래스
- 관련된 데이터와 기능을 그룹화합니다
- 상속 깊이는 최대 3단계를 넘지 않습니다
- 다중 상속은 신중하게 사용합니다

### 모듈
- 하나의 모듈은 하나의 책임만 가져야 합니다
- 모듈 크기는 500줄을 넘지 않도록 권장합니다

## 에러 처리

### 예외 처리
- 구체적인 예외를 명시합니다
- 빈 except 블록을 사용하지 않습니다
- 예외를 무시하지 않습니다
- 리소스 정리가 필요한 경우 finally를 사용하거나 context manager를 사용합니다

### Context Manager
- 파일, 네트워크 연결 등 리소스 관리 시 with 문을 사용합니다
- 자동으로 리소스가 정리되도록 합니다

### 사용자 정의 예외
- 의미 있는 예외 클래스를 정의합니다
- Exception 클래스를 상속받습니다
- 적절한 에러 메시지를 포함합니다

## 보안

### SQL 쿼리
- 문자열 포맷팅으로 SQL 쿼리를 생성하지 않습니다
- parameterized query를 사용합니다
- ORM 사용을 권장합니다

### 비밀번호
- 평문으로 저장하지 않습니다
- bcrypt, scrypt 등 안전한 해싱 알고리즘을 사용합니다
- 솔트를 사용합니다

### 민감 정보
- 코드에 API 키, 비밀번호 등을 하드코딩하지 않습니다
- 환경 변수를 사용합니다
- 설정 파일은 버전 관리에서 제외합니다

### 파일 경로
- 사용자 입력을 파일 경로에 직접 사용하지 않습니다
- 경로 검증을 수행합니다
- 허용된 디렉토리 내에서만 파일 접근을 허용합니다

### 입력 검증
- 모든 외부 입력은 검증해야 합니다
- 화이트리스트 방식을 우선 사용합니다
- 타입 체크를 수행합니다

## 성능

### 리스트 컴프리헨션
- 간단한 경우 for 루프 대신 리스트 컴프리헨션을 사용합니다
- 가독성이 떨어지는 경우 일반 루프를 사용합니다

### 제너레이터
- 큰 데이터셋 처리 시 제너레이터를 사용합니다
- 메모리 효율을 고려합니다

### 불필요한 연산
- 루프 내에서 반복적인 연산을 피합니다
- 결과를 캐싱하여 재사용합니다

## 타입 힌팅

### 타입 어노테이션
- 함수 시그니처에 타입 힌트를 추가합니다
- 복잡한 타입은 typing 모듈을 사용합니다
- 반환 타입을 명시합니다

### Optional과 Union
- None이 가능한 경우 Optional을 사용합니다
- 여러 타입이 가능한 경우 Union을 사용합니다

## 테스트

### 테스트 작성
- 모든 공개 함수에 대한 테스트를 작성합니다
- 테스트 함수명은 test_로 시작합니다
- 하나의 테스트는 하나의 기능만 검증합니다

### 테스트 구조
- Arrange-Act-Assert 패턴을 따릅니다
- 테스트 간 의존성을 제거합니다
- Mock을 적절히 활용합니다
